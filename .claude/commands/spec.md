코드 작성 전에 나한테 먼저 인터뷰를 진행해줘. 아래 항목을 하나씩 물어봐:

## 인터뷰 질문 목록

1. **프로젝트 개요**
   - 이 프로젝트/기능이 해결하려는 문제는?
   - 핵심 사용자는 누구인가?

2. **기술 스택**
   - 사용할 프레임워크/라이브러리
   - 데이터베이스 (있다면)
   - 배포 환경

3. **핵심 기능**
   - 반드시 포함될 기능 (MVP)
   - 나중에 추가할 기능 (Nice-to-have)

4. **UI/UX**
   - 디자인 참고 사이트나 스타일 있나?
   - 반응형 필요 여부
   - 컴포넌트 라이브러리 선호도

5. **비기능 요구사항**
   - 성능 목표 (응답 시간 등)
   - 보안 요구사항
   - 확장성 고려사항

## 인터뷰 완료 후

모든 답변을 바탕으로 `SPEC.md` 파일 생성:
- 프로젝트 개요
- 기술 스택 명세
- 기능 요구사항 (MVP / 확장)
- 아키텍처 다이어그램 (텍스트 형식)
- 파일 구조 제안
- 개발 순서 및 마일스톤

완료 후 "SPEC.md 생성 완료. Plan 모드(Shift+Tab)로 개발을 시작해보세요." 라고 안내해줘.
