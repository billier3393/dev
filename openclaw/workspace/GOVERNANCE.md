# GOVERNANCE.md — 거버넌스 및 승인 체계 (v11)

> 모든 구조적 변경은 동우님의 승인을 거쳐야 한다.

---

## 승인 프로세스

### 1. 팀 생성 프로세스
```
CEO → [팀 생성 제안서] → 동우님 → 승인/반려
```

**제안서 필수 항목:**
- 팀 이름 및 목적
- 필요 인원 (에이전트 수)
- 사용 모델 및 리소스 예산 (월 비용 추정 포함)
- 기존 팀과의 관계 (보고라인)
- 예상 토큰 소비량
- 생성 사유

**동우님 응답:**
- `승인` → CEO가 즉시 팀 생성, `teams/` 디렉토리에 기록
- `반려` → CEO가 사유 확인 후 수정 또는 철회
- `수정요청 [내용]` → CEO가 수정 후 재제안

### 2. 팀원 추가 / 팀 해산 / 구조변경
- 모두 동일한 승인 프로세스 적용
- 승인/반려는 `teams/governance_log.md`에 기록

### 3. 이메일 전송 승인
```
에이전트 → CEO → 동우님 → 승인/반려
```
- 수신자, 제목, 본문 미리보기 필수
- 동우님 `승인` 없이 절대 전송 금지

---

## 비상 권한
- **동우님 직접 명령**: CEO 거치지 않고 어떤 에이전트에든 직접 명령 가능
- **보안 에스컬레이션**: 보안팀은 중대 이슈 시 동우님에게 직접 보고 가능
- **긴급 정지**: `긴급정지` 명령으로 모든 에이전트 활동 즉시 중단

## 팀 확장 제한
- 동시 활성 에이전트: 최대 5개 권장 (토큰 효율)
- 팀 최대 인원: 리드 1 + 워커 4 (5명)
- 전체 팀 수: 제한 없음 (동우님 승인 시)
- 비활성 팀: 리소스 비할당 상태로 대기

## 정기 리뷰
- **주간**: COO가 팀 효율 + 예산 사용 보고서 → CEO → 동우님
- **월간**: CEO가 전체 조직 리뷰 + 비용 분석 → 동우님
