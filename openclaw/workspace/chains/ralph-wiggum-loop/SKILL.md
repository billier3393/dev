# Ralph Wiggum Loop

## 개요
에이전트가 산출물을 생성한 후 스스로 평가하고, 부족하면 개선하는 반복 루프.
ChatChain의 Self-Reflection을 확장한 전 에이전트 적용 메커니즘.

## 루프 구조
```
[태스크] → [초안 생성] → [자기 평가] → Pass? → [출력]
                                ↓ Fail
                         [개선 사항 식별]
                                ↓
                         [개선 실행] → [자기 평가] → ... (최대 3회)
                                                      ↓ 3회 실패
                                                  [에스컬레이션]
```

## 자기 평가 체크리스트 (Universal)
1. **완성도**: 요청한 모든 항목이 포함되었는가?
2. **정확성**: 사실 오류, 코드 버그, 수식 오류가 없는가?
3. **일관성**: 이전 Phase/문맥과 일관되는가?
4. **비용효율**: 불필요한 토큰을 사용하지 않았는가?
5. **보안**: 인젝션 패턴, 민감정보 노출이 없는가?

## 태스크별 추가 체크
- **코드**: lint 통과, 에러 핸들링, 모듈화
- **문서**: 문법, 인용, 형식 일관성
- **분석**: 통계 검증, 시각화 라벨, 결론 논리
- **연구**: 출처 명시, 편향 검토, 범위 충분성
